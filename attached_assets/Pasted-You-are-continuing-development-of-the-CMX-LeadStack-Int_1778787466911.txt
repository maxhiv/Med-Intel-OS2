--- 1C. Signal propagation job ---

Create a new service file:
artifacts/api-server/src/services/systemSignalPropagator.ts

Export a function: propagateSystemSignals()

Logic:
  1. Find all parent facilities (WHERE parentSystemId IS NULL AND facilityType = 'health_system')
  2. For each parent, find all child facilities (WHERE parentSystemId = parent.id)
  3. For each child, find all Tier 1 signals (signalType IN ('con_filed','con_approved',
     'bond_issuance','bond_issued','grant_awarded','nih_grant','rfp_posted'))
  4. For each Tier 1 signal on a child, upsert the SAME signal onto EVERY OTHER
     sibling facility with:
       - confidence = original.confidence * 0.80  (20% discount for indirect match)
       - metadata = { ...original.metadata, propagatedFrom: child.id, propagatedFromName: child.name }
       - signalType = 'system_signal_propagated' (add this to signalTypeEnum if not present)
     Use ON CONFLICT DO NOTHING on (facilityId, signalType, signalValue) to stay idempotent
  5. Also upsert the signal onto the parent facility itself with 0.90 confidence
  6. After propagation, call recomputeAllScores() so parent system scores reflect children

Register this in cron/index.ts to run daily at 07:00 (after the 06:00 ingest jobs):
  cron.schedule('0 7 * * *', async () => {
    await propagateSystemSignals()
  })

Also add POST /admin/signals/propagate-system as a manual trigger endpoint.

SECTION 2 — COMPOSITE LEAD CARDS
==========================================================================

WHY THIS MATTERS:
The current signals page shows a chronological list of individual signals.
A rep looking at it sees: "Grant awarded - St. Mary's - $450K" and then
20 rows later "CON PENDING - St. Mary's - MRI - $3.2M". They have to

SECTION 3 — FISCAL YEAR ALIGNMENT LAYER
==========================================================================

WHY THIS MATTERS:
A score-70 lead at 60 days before fiscal year end is worth 3x the rep 
attention of a score-80 lead at 15 days after FYE close. Capital equipment 
budgets are approved in Q3/Q4 of a hospital's fiscal year. If a rep reaches 
out after the budget is locked, they get pushed to "next year" and lose 12 
months. Most hospitals use Oct 1 FYE (federal fiscal year), but non-profits 
can use June 30 or Dec 31. This data is already in the CMS records we're 
ingesting — we're just not using it.

--- 3B. Populate FYE from CMS data ---

--- 3C. Time-adjusted score modifier ---

In signalScorer.ts, add a function: computeTimingBonus(facility):

  const now = new Date()
  const fye = facility.fiscalYearEndMonth
  if (!fye) return 0
  
  // Days until next FYE
  const currentYear = now.getFullYear()
  let fyeDate = new Date(currentYear, fye - 1, 1) // first day of FYE month
  if (fyeDate < now) fyeDate = new Date(currentYear + 1, fye - 1, 1)
  const daysUntilFYE = Math.floor((fyeDate.getTime() - now.getTime()) / 86400000)
  
  if (daysUntilFYE <= 30) return 20   // Budget decision happening NOW
  if (daysUntilFYE <= 60) return 15   // Active budget season
  if (daysUntilFYE <= 90) return 10   // Budget planning window
  if (daysUntilFYE <= 120) return 5   // Early positioning window
  return 0

Apply this bonus in computeFacilityScore() by adding computeTimingBonus() 
to the final score (cap total score at 100).

SECTION 4 — HCRIS COST REPORT INGESTOR
==========================================================================

WHY THIS MATTERS:
HCRIS (Healthcare Cost Report Information System) is published by CMS and 
contains every Medicare-certified hospital's audited financial statements — 
publicly available, no key required. The single most actionable number for 
capital equipment reps is the depreciation ratio: (medical equipment 
depreciation) / (total assets). When this ratio exceeds 40%, a facility's 
equipment is statistically in replacement cycle. No competitor is computing 
this from public data. It is the equivalent of a credit score for equipment 
replacement likelihood.

WHAT TO BUILD:

File: artifacts/api-server/src/services/hcrisIngestor.ts

Source: CMS HCRIS API
  https://data.cms.gov/provider-compliance/cost-report/hospital-provider-cost-report/api
(JSON REST, no API key, paginated via $limit/$offset SODA parameters)

Function signature: ingestHcris({ limit = 50 }: { limit?: number })
Returns: { signalsInserted: number, facilitiesUpdated: number, errors: number }

Implementation:

1. Fetch cost report records:
   GET https://data.cms.gov/resource/jcxf-uc9e.json?$limit=50&$offset=0
   (adjust the dataset ID if the SODA endpoint differs — check the API page)
   Sort by: $order=fy_end_dt DESC to get most recent reports first

2. For each record, attempt facility match:
   First: match by PRVDR_NUM against facilities.cmsId (exact)
   Second: if no match, try facilities.name ILIKE '%' || PROVIDER_NAME || '%'
   If no match: skip and increment errors counter.

3. Parse financial fields (field names vary by CMS version — use fallbacks):
   providerNum    = record.prvdr_num
   providerName   = record.provider_name || record.prvdr_name
   fyBegin        = record.fy_bgn_dt || record.fiscal_year_begin_date
   fyEnd          = record.fy_end_dt || record.fiscal_year_end_date
   totalAssets    = parseFloat(record.tot_assets_eoy_amt || '0')
   netPatientRev  = parseFloat(record.net_patient_rev_amt || '0')
   totalOpEx      = parseFloat(record.tot_operating_expense_amt || '0')
   
   For depreciation, try these fields in order:
     record.cap_depr_depl_amort_amt
     record.dep_depr_amort_and_depl
     record.total_depreciation_and_amortization
   Use the first non-zero value found. If none found, set depreciationAmt = 0.

4. Compute depreciation ratio:
   const depRatio = totalAssets > 0 ? depreciationAmt / totalAssets : 0

5. Upsert into financial_documents table:
   {
     facilityId: matched facility id,
     docType: 'hcris_cost_report',
     docDate: fyEnd,
     parsedJson: { providerNum, providerName, fyBegin, fyEnd,
                   totalAssets, netPatientRev, totalOpEx, 
                   depreciationAmt, depRatio },
     source: 'cms_hcris'
   }
   ON CONFLICT (facilityId, docType, docDate) DO UPDATE SET parsedJson = EXCLUDED.parsedJson

6. If depRatio > 0.40, insert purchase signal:
   {
     facilityId, 
     signalType: 'hcris_depreciation_spike',
     confidence: Math.min(0.95, 0.60 + depRatio),  -- scales from 60% to 95%
     signalValue: `hcris_depr_${providerNum}_${fyEnd}`,  -- idempotency key
     sourceUrl: 'https://data.cms.gov/provider-compliance/cost-report/hospital-provider-cost-report',
     metadata: { depRatio, depRatioPct: Math.round(depRatio * 100), fyEnd, providerNum }
   }
   ON CONFLICT (facilityId, signalType, signalValue) DO NOTHING

7. If depRatio > 0.40, also UPDATE facilities SET:
   fiscalYearEndMonth = new Date(fyEnd).getMonth() + 1,
   fiscalYearEndSource = 'hcris'
   (only if fiscalYearEndSource is null or 'hcris')

8. Add 250ms sleep between facility API calls to be polite.

Register in cron/index.ts at 06:00 alongside other ingestors.
Register in POST /signals/ingest/free-apis endpoint as 'hcris'.

Add 'hcris_depreciation_spike' to signalTypeEnum in lib/db/src/schema/enums.ts if not present.
Run db:generate then db:migrate.

Add to WEIGHTS map in signalScorer.ts:
  hcris_depreciation_spike: 25

Add cross-source bonus in signalScorer.ts:
  HCRIS depreciation spike + CON filed/approved: +10 bonus
  HCRIS depreciation spike + EOL equipment: +8 bonus
