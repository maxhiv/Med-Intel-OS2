Root Cause: Why APIs Aren't Working
THE PRIMARY ISSUE — No /api proxy in the frontend

The Vite dev server (artifacts/web/vite.config.ts) has no server.proxy configuration. The frontend makes calls to relative paths like /api/leads, /api/facilities, etc., which go to the Vite server on its port — but the Express API server is on a completely different port. Every API call gets a 404 from Vite.

The fix is a single block in vite.config.ts:

server: {
  port,
  proxy: {
    '/api': {
      target: `http://localhost:${process.env.API_PORT ?? 3001}`,
      changeOrigin: true,
    },
  },
  // ...rest unchanged
},
THE SECONDARY ISSUE — Platform admin account not seeded

If DATABASE_URL is set but the seed script (lib/db/src/seed.ts) hasn't been run, there's no hansen-holdings account in the database. The auth.ts middleware tries to auto-attach the platform admin to that account (WHERE slug = 'hansen-holdings') — if it doesn't exist, req.currentAccount is null, and requireAccount returns HTTP 403 on every protected route. Every page shows empty.

THE TERTIARY ISSUE — PLATFORM_ADMIN_EMAIL mismatch

auth.ts line 17 defaults to "max@hansenholdingsllc.com" but replit.nix sets PLATFORM_ADMIN_EMAIL = "maxhanseniv@gmail.com". If the Clerk account uses the Gmail address but the code defaults to the other, platform-admin privileges won't fire correctly.
